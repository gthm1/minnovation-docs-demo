# Internal Documentation (Demo)

This section represents internal-only content — architecture notes, deployment
runbooks, and support escalation info that should **not** be visible to clients.

:::danger Not actually access-restricted
In this demo, this section is served from the same public GitHub Pages site as the
public docs, so **it is not actually gated**. It exists to show the site *structure*
(a separate content instance, separate sidebar, separate URL path), not a working
access control.

To make this genuinely private, the underlying pages need to sit behind an
authentication layer at the hosting level — see the "Making this actually private"
section of the walkthrough guide for the options.
:::
